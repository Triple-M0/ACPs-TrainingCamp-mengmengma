# ACPs-TrainingCamp-mengmengma
# BLUETOOTH BEACON INDOOR POSITIONING SYSTEM
## 蓝牙 Beacon 室内定位系统

---

## PROJECT BACKGROUND
本项目是一套面向小范围室内场景的蓝牙信标定位原型系统，完整实现了从端侧蓝牙信号采集、服务端数据处理与位置解算，到前端位置可视化的全流程，是个人物联网与 Python 后端开发的学习实践项目。

---

## CORE FEATURES
- **端侧数据采集**：基于 OrangePi（香橙派）实现蓝牙 Beacon 设备的 RSSI 信号扫描、异常值过滤与数据上报
- **后端服务能力**：基于 Python Flask 搭建 RESTful API 接口，实现数据接收、存储、位置计算全流程自动化
- **定位算法实现**：基于对数距离路径损耗模型完成 RSSI 到距离的转换，针对 4×4 米小范围场景做了适配优化；支持最小二乘三边定位，同时针对 1-2 个扫描节点的场景做了加权质心算法降级兼容，保证定位可用性
- **数据平滑与边界处理**：通过历史位置加权平均实现定位结果平滑滤波，限制坐标在有效场景范围内，降低信号噪声带来的定位跳变
- **位置可视化**：提供前端页面，支持信标实时位置展示、历史轨迹查询
- **数据持久化**：基于 openGauss 数据库实现原始扫描数据与计算后的位置数据存储

---

## TECH STACK
- 后端开发：Python + Flask + psycopg2
- 数据库：openGauss
- 端侧硬件：OrangePi 开发板 + 蓝牙模块
- 核心算法：对数距离路径损耗模型、最小二乘三边定位、加权质心算法、滑动平均平滑滤波
- 前端：HTML + JavaScript

## 我对 ACPs 智能体互联网的理解
- 简单来说，ACPs就是智能体互联网的通用联网规则，就像TCP/IP协议——早年没有统一标准，不同品牌的电脑无法互通，而当下AI智能体领域，也正面临着同样的困境。


- 现在AI智能体开发普遍面临几个现实痛点：单个智能体能力有天花板，搞不定复杂任务；不同平台、框架开发的智能体互相“不认”，形成了大量信息孤岛；多智能体协作只能靠场景定制开发，成本高、复用性差。而ACPs和它要构建的智能体互联网，核心就是用一套开放、统一、可信的标准，让所有符合规范的智能体平等接入、互相识别、自由协作，最终形成可自组织、自协商的智能体协作网络。

- ACPs围绕智能体互联全流程，搭建了完整的标准化闭环：用AIC身份码、ACS能力描述规范，解决了智能体“你是谁、能干什么”的基础问题；通过ATR可信注册、AIA身份认证规范，构建了全链路可信准入体系，保障开放网络的安全；再以ADP发现协议帮智能体匹配协作伙伴，用AIP交互协议定好跨平台的“通用沟通语言”，靠DSP数据同步协议保证协作时的状态一致，让异构智能体也能顺畅配合。

- ACPs要做的智能体互联网，就是连通全世界的智能体，为AI时代的下一代互联网搭建核心底层基础设施。
